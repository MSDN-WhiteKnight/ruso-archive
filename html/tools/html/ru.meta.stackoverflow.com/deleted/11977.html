Объедниение меток WinUI 3 и Windows App SDK

Source - by aepot

Есть 2 метки, которые по сути про одно и то же.

winui3 https://ru.stackoverflow.com/tags/windows-app-sdk/info (4 вопроса)

windows-app-sdk https://ru.stackoverflow.com/tags/winui3/info (1 вопрос)

Первая про технологию нового Windows UI для разработчиков приложений, вторая - про SDK, позволяющий создавать приложения под этот UI. Обе сущности существуют в терминах Microsoft.

  • Библиотека пользовательского интерфейса Windows (WinUI) 3
  • Пакет SDK для приложений Windows

Я считаю, что вопросы, задаваемые здесь касаются все-таки больше инструмента Windows App SDK, чем непосредственно самой технологии WinUI 3.

Тег windows-app-sdk существует на английском Stackoverflow, тег winui3 не существует, но существует winui-3.

Проблема заключается в том, что вопросы, заданные в рамках по сути одной тематики, рандомно разбросаны по этим двум меткам.

Я предлагаю windows-app-sdk сделать основной меткой, а winui3 назначить её синонимом (или наоборот). Плюс переименовать метку winui3 в winui-3 для консистентности с английским SO.

Так как метки новые, я не имею достатчно репутации, чтобы внести синоним технически, поэтому прошу поддержки у сообщества, а так же хочу узнать ваше мнение по этому вопросу.
